Firestore 實戰：解決靜態網頁與動態資料庫的通訊難題

從 LocalStorage 到 Firestore，我們終於實現了真正的社群同步。

挑戰：靜態 HTML 如何安全連接資料庫？

我們遇到的最大問題是：在沒有後端伺服器（只有靜態 HTML）的情況下，如何安全地與資料庫通訊？

最初嘗試使用 `process.env` 在前端注入變數，但發現純靜態頁面無法讀取環境變數。最終解決方案是回歸本質：使用 Firebase 的客戶端 SDK，並配合嚴格的 Firestore 安全規則 (Security Rules)。

關鍵代碼：即時監聽 (onSnapshot)

onSnapshot(collection(db, 'ideas'), (snapshot) => {
    ideas = [];
    snapshot.forEach((doc) => {
        ideas.push({ id: doc.id, ...doc.data() });
    });
    renderIdeas(); // 數據一變，畫面秒更
});
Day 3
技術案例解析 2026-01-07

深度解析：如何讓網站具備「感知社群」的能力？

本案例展示了如何透過純前端技術，實現「社群行為驅動介面變化」的成長迴圈，是打造自動化產品的核心技術。

Step 1: 數據埋點 (LocalStorage)

做法：在用戶投票時，將資料寫入瀏覽器的 localStorage。

原理：利用客戶端存儲實現「無後端持久化」，降低開發成本，同時保證數據在重新整理後不丟失。

Step 2: 自動化篩選與監聽

做法：在首頁加載時，執行篩選函數，找出票數大於 5 的想法。

原理：透過 JS 的 filter() 陣列操作，實現數據的即時感知，並動態生成 DOM 元素。

Step 3: 視覺反饋與轉化

做法：將熱門想法顯示在首頁醒目位置，並加上「即將實現」標籤。

原理：建立「行為 -> 反饋」的正向激勵機制，讓訪客看到自己的投票真實影響了網站。

// 技術模板：前端自動監聽邏輯

const ideas = JSON.parse(localStorage.getItem('fl_ideas')) || [];
const hotIdeas = ideas.filter(i => i.votes > 5);
if (hotIdeas.length > 0) {
    renderHotSection(hotIdeas);
    triggerMilestoneAlert(); // 觸發里程碑通知
}

🎓 學習總結

  • 不需要複雜的後端，前端也能實現「動態感」。
  • 數據存儲位置不重要，重要的是如何讓數據「流動」起來。
  • 建議練習：嘗試為你的網站加入一個「最近訪客記錄」功能，同樣使用 LocalStorage。

深度解析：權力下放 - 用 LocalStorage 打造去中心化投票

FL 不再是我一個人的獨角戲。我們實作了基於 localStorage 的投票系統，這意味著網站的發展方向現在掌握在每一位訪客手中。

Step 1: UI 設計 (Tailwind)

做法：使用 Tailwind CSS 快速構建投票卡片，包含排名、標題和按鈕。

<div class="flex items-center justify-between p-4 border rounded-lg hover:bg-green-50 transition">...</div>

Step 2: 邏輯實現 (JS + LocalStorage)

做法：編寫 JavaScript 函數來處理投票點擊，更新數據陣列，並保存到 localStorage。

// 技術模板：LocalStorage 投票邏輯

function vote(id) {
    const idea = ideas.find(i => i.id === id);
    if (idea) {
        idea.votes++;
        saveData(); // localStorage.setItem(...)
        renderIdeas();
    }
}

Step 3: 防刷票機制

做法：在 localStorage 中記錄已投票的 ID 列表，防止重複投票。

原理：雖然這不是絕對安全的（用戶可以清除緩存），但對於一個實驗性專案來說，這是一個足夠輕量且有效的解決方案。

🎓 學習總結與應用建議

  • 去中心化思維：不一定需要後端才能做應用。
  • 用戶賦權：讓用戶參與決策能顯著提升參與感。
  • 應用建議：試著在你的個人網站加上「喜歡」按鈕，記錄在本地。
